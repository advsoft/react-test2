## Авторизация

Задание предполагает использование dummyjson API. В общем случае хранить токен можено как в куках (безопаснее)
так и в local/session storage (менее безопасно, также определенные проблемы при SSR). Получить от dummyjson
сессионные куки без MaxAge и Expires судя по всему невозможно, поэтому в рамках этого задания храним
в session или local storage.

После успешного логина токен и данные пользователя сохраняются в зависимости от чекбокса «Запомнить меня»:

- **С галочкой** — `localStorage` (ключ `auth_token`). Сессия сохраняется между закрытием браузера и перезагрузкой.
- **Без галочки** — `sessionStorage` (тот же ключ). Сессия сбрасывается при закрытии вкладки.

Флаг «Запомнить меня» хранится в `localStorage` под ключом `auth_remember_me`.

При инициализации приложения (`authStore`) выбирается хранилище по флагу `auth_remember_me`, затем из него читается `auth_token`. Если флаг `true` — читаем из `localStorage`, иначе — из `sessionStorage`.

При выходе очищаются оба хранилища и флаг `auth_remember_me`.

## Продукты

Продукты в таблицу не добавляем т.к. нет логики API добавления.

## State management

Для серверного состояния - TanStack Query. Упрощает код, в каких-то случаях кеш будет работать сокращая количество 
запросов к BE.

Для клиенсткого состояния - Jotai. В данном проекте client state очень мал и jotai тут отлично вписывается. Хотя в целом 
можно и вообщего без него.

## UI
